# Refrain 产品总规格（草案）

- 状态：规格核对中
- 版本：0.1
- 日期：2026-07-13
- 参考：Foobox 8 系列布局，运行于 foobar2000 2.25.10
- 本文作用：定义产品边界、完整模块和人类可理解的程序逻辑；本文未获用户批准前不得开始功能实现

## 1. 产品目标

Refrain 是一个面向 Windows 10/11 x64 和 foobar2000 2.25.10 的原生 Columns UI 组件。它以原生 C++、Win32、Direct2D 和 DirectWrite 重建用户熟悉的 Foobox 布局与交互，使主要界面不再依赖 JavaScript UI 宿主，同时保留 foobar2000 的播放、媒体库、播放列表和菜单能力。

“重建”指用户能以熟悉的方式完成日常操作，并且在空数据、错误、窗口变化和重启后仍能稳定工作；它不指逐行翻译 Foobox 脚本，也不继承 JSplitter 的架构。

## 2. 目标用户和主要场景

目标用户是已经习惯 Foobox，但不想继续依赖 JavaScript 面板宿主的 foobar2000 用户。主要场景包括：

1. 启动 foobar2000 后直接进入稳定、熟悉的整合界面。
2. 浏览媒体库封面或分类，找到音乐并送入指定播放列表。
3. 创建、选择、重命名、整理和删除播放列表。
4. 在带分组、封面和可配置列的播放列表中选择、排序、拖放和播放曲目。
5. 查看当前曲目、播放进度、音量、输出设备、播放顺序、封面和适用的扩展信息。
6. 调整面板比例、主题和列表显示方式，并在下次启动恢复。
7. 在没有封面、没有媒体库、播放停止或文件失效时仍得到清楚且可恢复的界面反馈。

## 3. 本规格的完整可用边界

本规格把产品拆成多个纵向模块，是为了便于检查先后，不是允许交付残缺模块。每个模块进入实现后都必须达到其完整可用条件。

### 3.1 原生界面外壳

负责承载 Refrain 的全部区域，分配空间，处理 DPI、主题、窗口大小变化、绘图资源和输入焦点。它必须能在 Columns UI 中被稳定创建、销毁和重新创建；不能要求 JavaScript 宿主存在。

完整可用条件：正常启动和卸载；最小/最大可用尺寸行为明确；拖动调整流畅；DPI 变化后尺寸和文字正确；设备丢失后能恢复绘制；窗口关闭时没有悬挂回调或资源。

### 3.2 顶部菜单、标题与窗口控制

提供与参考布局对应的主菜单入口、当前曲目信息或窗口标题，以及适用的最小化、最大化/还原和关闭操作。是否由组件直接控制顶层窗口，必须先通过 Columns UI SDK 能力验证；若 SDK 不允许，需提出有证据的等价布局并由用户批准。

完整可用条件：菜单内容来自 foobar2000 正式菜单服务；窗口按钮状态与真实窗口一致；双击、拖动和系统缩放行为无冲突；键盘菜单和系统菜单仍可使用。

### 3.3 播放控制带

包含打开、上一首、播放/暂停、下一首、停止、当前时间、总时长或剩余时间、可拖动进度、媒体库入口、播放顺序、输出设备、静音和音量。

完整可用条件：所有按钮与 foobar2000 实际状态同步；不能跳转的流显示不可跳转状态；拖动进度时反馈明确；音量与静音变化双向同步；输出设备和播放顺序菜单反映当前选项；播放停止、暂停和换曲时没有旧状态残留。

### 3.4 工作区导航与布局

负责在媒体库、播放列表管理、播放列表、封面/信息以及经批准的扩展面板间切换，并管理横向或纵向分隔区。布局选择、分隔位置、可见区域和上次使用页面需要持久化。

完整可用条件：所有目标页面可达；切换不会丢失选择和滚动位置；分隔条有最小尺寸限制；窗口变窄时有已批准的收缩策略；重启后恢复；配置损坏时回到安全默认布局。

### 3.5 媒体库封面浏览

按已批准的分类方式整理媒体库，显示封面网格或列表，支持筛选、键盘定位、滚动、选择、双击播放或发送到播放列表，以及适用的拖放和右键命令。

完整可用条件：大型媒体库仍可操作；封面异步加载且不会阻塞绘图；缺图有一致占位；媒体库变化可增量更新或安全重建；搜索为空、无媒体库、文件失效和加载失败都有明确状态；缓存有容量和失效规则。

### 3.6 播放列表管理

显示普通播放列表和自动播放列表，支持选择、新建、重命名、复制、删除、排序或拖动，以及把曲目放入目标列表。危险操作需要符合参考习惯的确认方式，且不得错误删除自动列表或系统保留列表。

完整可用条件：列表变化实时同步；重名、空名、取消重命名、删除当前列表、只读/自动列表和外部拖入行为明确；键盘和右键操作完整；删除确认不会因异步状态变化作用到错误对象。

### 3.7 分组播放列表

显示当前播放列表，支持参考中的专辑/艺术家/流派/目录等分组、可配置列、封面、排序、当前播放标记、选择、焦点、队列状态、评分/心情、拖放和上下文菜单。

完整可用条件：普通和大列表可流畅滚动；选择与焦点遵循 Windows 列表习惯；列宽、顺序、显示状态和分组方式持久化；标题格式计算失败时安全回退；队列、播放状态和元数据变化准确刷新；内部与外部拖放不会丢曲或误排序。

### 3.8 封面、曲目信息与扩展内容

显示正在播放项或当前焦点项的封面和元数据，并按批准范围承载歌词、Biography 或可视化内容。核心封面与基础元数据由 Refrain 原生提供；任何外部组件依赖都必须单独批准并定义缺失时行为。

完整可用条件：跟随模式可见且可切换；异步封面晚到时不会覆盖新曲目；支持缺图、多个封面来源和刷新；播放停止和无选择时显示安全状态；扩展组件缺失或失败不影响其他界面。

### 3.9 搜索与属性

提供参考布局中可达的快速搜索和曲目属性入口。搜索结果、作用范围和执行动作必须明确；属性编辑优先调用 foobar2000 正式服务，而不是自行改写标签文件。

完整可用条件：空查询、无结果、特殊字符、输入法、取消和大量结果行为稳定；属性窗口针对正确选择打开；保存、失败和外部元数据变化能正确反馈。

### 3.10 主题、字体与设置

提供与 Foobox 参考相符的深浅外观、强调色、字体、列表布局和行为设置。系统主题、Columns UI 颜色和 Refrain 自定义设置之间的优先关系必须在模块规格中固定。

完整可用条件：设置即时预览或明确提示重启；取消不会误保存；恢复默认可用；所有配置有稳定标识和版本迁移；非法旧值不会导致崩溃或不可见文字；高对比度和系统字体缩放有明确策略。

## 4. 整体程序逻辑（人类语言）

### 4.1 启动

1. foobar2000 让 Columns UI 创建 Refrain 界面。
2. Refrain 先建立只描述当前界面状态的内存模型，不立即在绘图过程中读取磁盘。
3. 程序读取自己的版本化设置；每个值都要检查是否合法，损坏或过期的值使用安全默认值并记录一次可理解的诊断。
4. 程序连接 foobar2000 的播放、播放列表、媒体库、颜色、字体和菜单服务，并取得第一份状态快照。
5. 界面按照当前 DPI 和窗口大小计算布局，创建绘图资源后显示。
6. 封面和大型列表数据在后台准备；结果回到界面线程前必须确认目标窗口和目标曲目仍然有效。

### 4.2 用户操作

1. 鼠标、键盘或拖放先被转换成“用户想做什么”，例如播放下一首或选择某个播放列表。
2. 状态模型判断当前是否允许该操作；不允许时显示原因或禁用入口，而不是静默失败。
3. 允许的操作交给 foobar2000 正式服务执行，Refrain 不伪造已经成功的界面状态。
4. foobar2000 回调确认状态变化后，模型更新；界面只重绘受影响区域。
5. 需要保存的用户偏好采用延迟合并写入，窗口关闭前安全完成；音乐库和播放列表数据本身继续由 foobar2000 管理。

### 4.3 异步封面和大型数据

1. 界面请求封面时附带“请求对应哪首曲目/哪个分组”的身份标记。
2. 先检查内存缓存，再通过 foobar2000 服务异步取得图片；绘图线程不等待磁盘。
3. 结果回来时再次比较身份标记、窗口生命期和当前布局；过时结果直接丢弃。
4. 图片按 DPI 和显示区域生成合适尺寸的绘图资源，原始数据与设备相关资源分开管理。
5. 缺图和失败进入有上限的重试/失效规则，不能无限循环或持续刷日志。

### 4.4 关闭、卸载和异常恢复

1. 先标记界面正在关闭，停止接受新的异步显示请求。
2. 注销 foobar2000 回调和计时器，确保晚到结果只会被丢弃。
3. 保存有效设置并释放绘图、窗口和系统资源。
4. 如果 Direct2D 设备丢失，只重建设备相关资源，不丢失用户选择或列表状态。
5. 单个封面、标题格式或扩展面板失败不能拖垮整个组件。

## 5. 数据归属

| 数据 | 谁负责保存 | Refrain 的责任 |
| --- | --- | --- |
| 音频文件和标签 | 用户与 foobar2000 | 通过正式服务读取/编辑，不直接假定文件可写 |
| 媒体库与播放列表 | foobar2000 | 订阅变化、发出合法操作、显示结果 |
| 播放状态、音量、输出、顺序 | foobar2000 | 双向同步，不维护冲突副本 |
| 布局、列、分组、主题偏好 | Refrain | 版本化持久化、校验、迁移、恢复默认 |
| 封面缓存 | Refrain 或 foobar2000 服务 | 有容量、失效和清理规则，不作为唯一数据来源 |
| 参考 Foobox 配置 | 只读参考 | 不写入、不直接作为运行配置加载 |

## 6. 性能与稳定性目标

- 绘图和输入处理不得执行阻塞式文件或网络 I/O。
- 滚动、拖动进度和调整分隔条时，目标是在常见硬件上保持视觉连续，不因逐项同步加载封面产生停顿。
- 列表只绘制当前可见部分，大型媒体库不一次为所有项目创建窗口控件或完整尺寸图片。
- 组件卸载、窗口反复创建销毁、播放快速切歌和媒体库批量变化必须作为专门测试场景。
- 所有资源数量必须有上限或明确释放时机；缓存不能随使用时间无限增长。
- Debug 与 Release x64 都要构建；任何新增警告必须消除或由用户批准原因。

具体可量化的启动时间、帧时间、媒体库规模和内存上限，需要在取得开发机基线后写入各模块规格，不能在没有测量时编造数字。

## 7. 安全、隐私与依赖

- 不记录音频路径、搜索文字或媒体库内容到无边界日志。
- 不执行来自标签或配置的命令；显示文本按文本处理。
- 不向网络发送媒体信息，除非用户批准一个明确的网络功能及其隐私说明。
- 未经批准不新增运行时 DLL、服务、脚本引擎或外部组件依赖。
- 第三方源码和二进制必须记录来源、版本、许可证和再分发条件。

## 8. 明确不属于当前已批准范围的内容

以下不是永久排除，而是在用户批准前不能擅自加入：移动端/非 Windows 平台、云同步、在线账户、音乐商店、自动下载歌词或艺人资料、音频解码/DSP、修改 foobar2000 核心文件、兼容 32 位 foobar2000。

## 9. 必须共同核对的问题

- [ ] 产品形态是否就是一个整合式 Columns UI 组件/根面板，还是需要多个可独立放置的面板同时提供。
- [ ] 第一套精确参考布局选择哪个 Foobox `.fth` 文件。
- [ ] 顶部自绘标题栏和窗口按钮是否是必须一比一还原的核心行为。
- [ ] 媒体库默认入口选择 Album List、ReFacets，还是 Refrain 自己的封面浏览。
- [ ] 歌词、Biography、频谱是否属于第一套完整可用范围，以及能否依赖现有外部组件。
- [ ] 评分和心情字段的实际写入规则、用户平时使用方式和所需组件。
- [ ] 深色、浅色和强调色的默认值与切换入口。
- [ ] 参考中的哪些右键菜单、键盘快捷键和拖放操作是日常必需，必须优先实际观察。
