# Refrain UI/UX 目标清单（草案）

- 状态：规格核对中
- 版本：0.1
- 日期：2026-07-13
- 本文作用：列出可见界面、交互状态和后续参考观察目标；尺寸与颜色尚未实际测量的地方不写伪精确数值

## 1. 总体体验目标

- 熟悉：Foobox 用户不需要重新学习日常播放、浏览和管理路径。
- 原生：文字、焦点、键盘、菜单、拖放、DPI 和辅助功能遵循 Windows/foobar2000 的稳定行为。
- 清楚：可点击、悬停、按下、选中、当前播放、禁用、加载和失败状态可以被区分。
- 连续：切换页面、滚动列表、拖动进度和调整分隔条时不闪烁、不跳回、不丢选择。
- 可恢复：误操作有适当确认或撤销路径，设置可恢复默认，配置损坏不会让界面不可用。
- 忠实：第一套主题先复现用户选定的参考布局，不主动“现代化重设计”；差异必须说明原因并经用户同意。

## 3. 布局目标

### 3.0 用户截图确认的主界面基准

两张用户提供截图描述同一主界面的两种右下状态，作为 `foobox8 basic.fth` 的首个可见结构基准。结构由上到下、由左到右如下：

1. 顶部保留 Windows 原生标题栏；foobar2000 主菜单与当前播放摘要位于其下方的顶部区域。截图中的隐藏/自绘标题栏方式不复刻。
2. 左栏顶部为当前播放列表搜索和新增入口，其下为筛选、媒体库入口、用户已有普通/自动播放列表及各自项目数；当前列表用强调色整行标记。
3. 中栏为带表头的分组播放列表。截图可见封面、序号、标题、艺术家、专辑、流派、等级、比特率、编码和时间列；分组标题行包含专辑/艺术家摘要和年份，组内可显示封面。
4. 右栏上部为正方形封面，其下依次为五星评分、歌曲标题、“歌手 | 专辑”和“编码 | 编码配置 | 比特率 | 采样率”两条次要信息。两条次要信息同时显示，不使用 Foobox 默认的 12 秒轮换。
5. 右栏下部默认显示嵌入的 ESLyric 歌词；鼠标中键切换为截图二中的字段名—字段值歌曲信息列表，再次中键返回歌词。歌词面板内的双击不由 Refrain 截获，继续触发 ESLyric 自身的全屏歌词行为。
6. 底部控制区上沿是一条横跨窗口的进度条，左右显示已播放时间和总时长；下方左组为截图中的功能入口，中组为播放顺序/上一首/播放暂停/下一首/停止，右组为列表与播放相关入口、静音和音量。
7. 截图中的按钮位置、组间关系和视觉层级均作为布局约束；暂时无法仅凭图标确认的命令含义必须通过参考界面实际操作确认，不能按图形猜测功能。
8. 不在主界面放置 Biography 或频谱区域。用户另行安装频谱组件时，foobar2000 自身菜单仍可访问它，但 Refrain 不为其增加按钮或默认启动行为。

### 3.1 尺寸与缩放

- 所有设计尺寸以与 DPI 无关的逻辑单位表达，不能把参考截图像素直接写死。
- 字号、图标、命中区域、分隔条和圆角/线宽按 DPI 一致缩放；显示器切换时无需重启。
- 每个可伸缩区域都要规定最小可用尺寸。达到最小值后采用经过批准的隐藏、折叠或滚动策略，不能让控件互相覆盖。
- 用户拖动的分隔位置以逻辑比例或可迁移逻辑尺寸保存；更换 DPI 或窗口尺寸后仍合理。
- 长标题、长艺术家、中文、英文、数字和无空格文本都要定义截断与提示行为。

### 3.2 视觉层级

- 当前播放、当前选中、键盘焦点和鼠标悬停是不同概念，不能只靠一种相同背景色表示。
- 主操作（播放/暂停）应比相邻传输操作更容易定位，但不改变参考布局的基本节奏。
- 强调色只用于当前状态、选中或进度等有意义位置，不能造成整屏竞争。
- 分隔线、次要文字和禁用图标在深浅主题中均需保持足够辨识度。
- 封面加载、缺图和加载失败采用稳定占位，不导致列表行高或布局跳动。

### 3.3 主题

- 第一套主题的背景、文字、强调色、分隔线、选择色和按钮状态从用户指定的 Foobox 布局实际取样。
- 深色与浅色如果同时纳入首个完整范围，两者都必须逐状态验收，不能仅反转颜色。
- 系统高对比度、Columns UI 颜色和 Refrain 自定义色的优先级要在主题模块规格中说明。
- 颜色不能是传达播放、错误或选择状态的唯一方式；同时使用图标、文字或结构提示。

## 4. 输入与交互目标

### 4.1 鼠标

- 所有图标按钮都有稳定命中区域和简短中文或跟随 foobar2000 语言的提示。
- 按下后移出再松开应取消激活，除非该控件明确是拖动操作。
- 双击只用于参考中已经确认的高频操作，且不能与单击选择产生不可逆冲突。
- 右键菜单针对鼠标所在对象或当前选择的规则必须一致，菜单打开后目标对象不能因异步刷新悄悄改变。
- 鼠标滚轮优先滚动指针所在列表；在音量或进度上的滚轮行为需实际观察后决定，避免误调。

### 4.2 键盘

- Tab/Shift+Tab 能按可预测顺序移动焦点；焦点始终可见。
- 列表支持方向键、Page Up/Down、Home/End、Enter、空格以及适用的 Ctrl/Shift 多选。
- 重命名支持 Enter 确认、Esc 取消；删除等危险快捷键遵循确认规则。
- 菜单和 foobar2000 全局快捷键不能被 Refrain 无条件吞掉。
- 搜索支持中文输入法组合状态，组合文字未提交前不能错误触发快捷键。

### 4.3 拖放与分隔条

- 拖动开始、允许放置、禁止放置、插入位置和目标播放列表都有可见反馈。
- 拖放失败不得删除来源内容；跨列表复制或移动规则必须和 foobar2000 一致。
- 分隔条悬停时改变指针并显示可拖动性，拖动中连续反馈，双击复位与否需参考确认。
- 拖动窗口外、按 Esc、窗口失焦或目标被删除时都能安全取消。

## 5. 状态设计清单

每个用户可见模块在实现前都必须填写下表所列状态，不能只画“有数据且成功”的一张界面。

| 状态 | 必须回答的问题 |
| --- | --- |
| 初始加载 | 显示什么，是否可操作，如何避免闪烁？ |
| 正常有数据 | 主信息、次信息和主要操作是什么？ |
| 空数据 | 为什么为空，用户下一步能做什么？ |
| 加载中 | 哪个局部在加载，其他区域是否仍可用？ |
| 失败 | 发生什么、影响哪里、能否重试或恢复？ |
| 禁用 | 为什么当前不能操作，是否需要提示？ |
| 悬停/按下 | 命中区域和即时反馈是什么？ |
| 选中/焦点 | 鼠标选择、键盘焦点、当前播放如何区分？ |
| 数据变化 | 外部修改、换曲或列表刷新时如何保持上下文？ |
| 关闭/切换 | 未完成异步请求和临时编辑如何处理？ |

## 6. 关键区域的目标行为

### 6.1 播放控制带

- 播放按钮根据“正在播放、暂停、停止”显示正确图形和提示。
- 时间文本格式、剩余时间显示方式和点击切换行为以参考实际观察为准。
- 用户拖动进度时先显示预览位置，松开后请求跳转；foobar2000 确认前不伪造永久状态。
- 网络流或不可跳转内容显示禁用进度操作，但仍显示可取得的时间信息。
- 音量条同时支持点击、拖动和键盘微调；静音前音量及解除静音规则与 foobar2000 状态一致。

### 6.2 媒体库入口

- 主界面入口调用 foobar2000 原生 Album List，不在 Refrain 内绘制封面网格或维护另一份媒体库分类。
- Album List 可用时遵循 foobar2000 自身的窗口、键盘、搜索和发送到播放列表行为，Refrain 不改写用户已经习惯的操作。
- 媒体库未启用或命令不可用时，入口保持可理解的禁用或错误反馈，不能静默无响应。
- Refrain 升级不修改 Album List 的视图、查询或 foobar2000 已保存的媒体库配置。

### 6.3 播放列表管理

- 普通列表、自动列表和当前活动列表有明确区分。
- 新建后直接进入可取消的重命名；名称校验失败时保留编辑内容并说明原因。
- 删除前确认对象名称；确认期间列表改变时仍通过稳定身份识别原对象。
- 拖入曲目时高亮目标列表；拖动列表排序时显示插入位置。

### 6.4 分组播放列表

- 表头列可以按批准范围显示/隐藏、调整宽度和排序，设置重启后保留。
- 分组标题与曲目行的层级、封面占位和折叠状态清楚；折叠状态是否持久化需参考确认。
- 当前播放行、队列序号、选中行、焦点行和悬停行不互相覆盖语义。
- 多选、范围选择、拖放、右键命令、键盘播放和滚动定位遵循一致的 Windows 列表习惯。
- 元数据或播放统计更新只刷新受影响行，并保持滚动位置和选择。

### 6.5 封面、歌词与歌曲信息区

- 明确显示当前跟随“正在播放”还是“列表焦点”；用户切换后有持久、可理解的状态。
- 换曲时旧封面可以短暂保留还是立即显示加载占位，需要通过参考观察决定。
- 无封面提供默认图和可用的刷新/打开来源命令。
- 右上始终保留封面、评分、标题和两条次要信息，不因右下视图切换而改变尺寸或内容来源。
- 右下默认显示 ESLyric；缺失时在相同区域显示安装提示，不让右栏或主播放列表坍塌。
- 鼠标中键只在“歌词”和“只读歌曲信息”之间切换，不打开新窗口；切换状态重启后恢复。
- 双击歌词区域交给 ESLyric 处理，以使用其自身全屏界面；双击歌曲信息区域不触发全屏歌词。
- 歌曲信息按字段名和值排列，长值可选择、复制和横向处理，不直接在列表内编辑。

## 7. 可访问性与可理解性

- 自绘交互元素必须提供键盘焦点、可访问名称、角色和当前状态，不能只是一块会响应鼠标的像素区域。
- 文本缩放后不裁掉关键按钮或状态；必要时允许布局重排。
- 动画只用于解释状态变化，时间短且可受系统“减少动画”偏好影响；没有动画也不影响理解。
- 错误信息用人类语言说明影响和恢复动作，不向普通用户直接倾倒 HRESULT 或 C++ 异常文本。
- 提示文本简短稳定；复杂设置在规格和验收文档中提供中文解释。

## 8. 参考界面观察清单

在确定任何像素、颜色或精确交互前，需要在用户指定的 Foobox 主题中完成以下观察并保存截图/记录：

- [ ] 默认启动、正在播放、暂停、停止四种整体界面。
- [ ] 普通窗口、最窄可用窗口、最大化和不同 DPI。
- [ ] 顶部菜单、标题拖动、窗口按钮的悬停/按下/双击/右键。
- [ ] 每个播放按钮、进度、时间、播放顺序、输出、静音和音量。
- [ ] 左栏搜索、筛选、Album List 入口和播放列表管理的全部按钮。
- [ ] 播放列表的新建、重命名、删除、自动列表和拖放。
- [ ] 播放列表分组、列、排序、多选、队列、评分/心情和空列表。
- [ ] 封面跟随、缺图、多图、刷新、歌词/歌曲信息中键切换和 ESLyric 双击全屏。
- [ ] 深色/浅色、强调色、字体和设置恢复默认。
- [ ] 无媒体库、无播放列表、网络流、文件失效和外部修改。
