# 稳定性、性能与生命周期验证规格

- 版本：0.1
- 状态：已批准，可执行
- 对应任务：[`../../tasks/019-完成稳定性性能与生命周期验证/README.md`](../../tasks/019-完成稳定性性能与生命周期验证/README.md)
- 最后更新：2026-07-15

## 1. 目标

在不增加产品功能的前提下，对 Refrain 当前 0.1 功能进行可重复的短时稳定性、资源和生命周期验证。当前任务只对能够自动检查或在一次受控会话内复现的结论负责；需要数天或数周日常使用才能判断的缓慢增长、罕见设备错误和第三方组件组合问题，登记为 1.0 发布后的持续观察项。

## 2. 当前可以验证

1. x64 Debug/Release 从当前源码完整构建，全部模型测试在两套配置下通过。
2. 对大播放列表、大分组、大专辑来源和重复设置序列执行确定性压力输入，检查越界、异常退出和无上限容器。
3. 重复运行测试进程，检查退出、返回码、单轮耗时和进程残留。
4. 审计主面板的线程、计时器、foobar2000 回调、OLE 拖放、ESLyric 子窗口、Direct2D/DirectWrite/GDI 资源、异步封面请求和缓存上限。
5. 在隔离 `foobar-dev` 中进行短时启动、空闲、播放和退出基线；记录 Working Set、Private Bytes、CPU、线程数、Handle、GDI/USER 对象的变化，不用单张任务管理器截图下结论。
6. 审计 Release 包内容、DLL/安装包哈希和 Release 日志边界。

## 3. 1.0 后持续观察

- 数天连续播放后是否存在缓慢内存增长。
- 罕见音频输出设备休眠、热插拔或驱动异常。
- 网络歌词服务、媒体库监控和第三方组件在长期运行中的偶发故障。
- 极低概率的设备丢失、系统睡眠/唤醒、远程桌面和多显示器组合。
- 用户真实媒体库中特殊标签、损坏图片和文件系统权限组合。

这些项目不阻止当前任务完成自动验证，但必须保留为发布后观察清单；出现可复现证据时再建立修复任务。

## 4. 判定方法

- 资源判断以同一构建、同一实例、同一操作序列的增量和是否趋于稳定为主，不以不同程序的一次瞬时 Working Set 比较为结论。
- UI 线程不得执行封面文件阻塞读取；异步结果必须带代次或身份检查，旧结果不得覆盖新目标。
- 缓存必须有数量或字节上限，并在来源切换、窗口销毁和组件卸载时释放或失效。
- 每个创建的线程、计时器、回调、拖放注册、子窗口和图形资源必须存在对称退出路径。
- 自动压力用例不得依赖用户日常安装、C 盘 foobar2000 或只读参考目录。
- 性能阈值先记录本机基线；只有能稳定复现并明显超出基线波动的退化才修改代码，避免为了漂亮数字破坏已验收行为。

## 5. 明确不做

- 不加入 CoverFlow、迷你播放器或其他新 UI。
- 不在本任务实现延期的完整键盘导航和屏幕阅读器对象树。
- 不触碰日常 foobar2000、C 盘安装或 `D:\Dev\foobar2000` 参考配置。
- 不用强制结束进程冒充正常退出验证。
- 不承诺当前短时测试能够证明数周运行绝无泄漏。

## 6. 当前任务通过标准

- Debug/Release 构建与全部自动测试通过。
- 大输入和重复循环无崩溃、越界、超时或残留测试进程。
- 静态生命周期审计没有未解释的无界缓存、悬挂线程、未注销回调或 Release 隐私日志。
- 隔离实例可正常启动和退出；短时资源数据已记录，并明确区分测量值与推断。
- 所有可复现问题已经修复并复测；无法短时证明的项目进入 1.0 后观察清单。
