# Playback Shell 模块规格

- 状态：已验收
- 对应路线：`04 实现原生主界面外壳与播放控制`
- 最后更新：2026-07-13

## 1. 用户可见结果

Columns UI 的布局编辑器中出现一个名为 `FooCrate`、分类为 `Panels` 的多实例面板。面板使用原生 Win32 子窗口和 Direct2D/DirectWrite 绘制，不创建顶级窗口，因此 foobar2000 的 Windows 标题栏、系统按钮和菜单保持不变。

本步骤只显示已经可工作的 `Playback Bar`。上行是已播放时间、`Seek Bar` 和总时长；下行中央是 Previous、Play/Pause、Next、Stop，右侧是 Mute 和 Volume。尚未实现的 Playlist Browser、Playlist View、Now Playing Panel、Workspace Switcher 和其他 Playback Tools 不显示边框、标题、假数据或假按钮。

## 2. 状态来源与同步

- 播放、暂停、停止、当前位置、总时长、能否跳转和音量均从 foobar2000 `playback_control` 读取。
- 面板创建时立即读取当前真实状态；创建之前已经播放也必须显示正确。
- 订阅播放开始、新曲目、停止、暂停、跳转、时间和音量回调；回调只更新状态并重绘受影响窗口。
- 多个 FooCrate 面板允许同时存在，全部观察同一个 foobar2000 播放状态；操作任意实例后其他实例同步更新。

## 3. 操作规则

- Previous/Next 使用 foobar2000 的用户语义：上一首在适用时先回到当前曲目开头，下一首遵循当前播放顺序。
- Play 在停止时开始当前播放列表，在播放时切换 Pause/Resume；图形随真实暂停状态变化。
- Stop 仅在正在播放时启用。
- 可跳转且长度有效时，按下进度条进入预览；移动时只改变预览时间，松开且仍在进度条命中范围内才调用一次跳转。Esc、失焦、捕获丢失或移出后松开均取消，不伪造跳转成功。
- 网络流、长度未知或 foobar2000 报告不可跳转时，进度条禁用；时间仍按可取得的信息显示，未知总长显示 `--:--`。
- 音量条采用 Columns UI 官方 Volume toolbar 的同一非线性换算：滑块比例到分贝为 `10 × log₂(比例)`，反向为 `2^(分贝/10)`，并在 `-100 dB` 钳制。这样 FooCrate 与 Columns UI 控件宽度不同时，同一真实音量仍处于基本一致的相对位置；拖动期间向 foobar2000 发出真实音量值，最终状态由音量回调确认。
- Mute 调用 foobar2000 静音切换。静音时保留音量位置并显示静音状态；输出设备使用自定义音量模式时不提供可直接跳到最大值的滑杆，只保留逐步音量/静音能力并显示 `Device volume`。

## 4. 输入、DPI 与生命周期

- 所有命中尺寸按当前窗口 DPI 从 DIP 换算；处理 `WM_DPICHANGED` 后重新布局并重建文字资源。
- 按钮支持悬停、按下、移出取消、工具提示文本和键盘焦点；Space/Enter 激活焦点按钮，Tab 交给 Columns UI/foobar2000 的导航与快捷键链。
- 面板只在 foobar2000 主线程调用播放服务。销毁时先注销回调、释放鼠标捕获和绘图资源，再销毁窗口；销毁后的消息或回调不得访问窗口。
- Direct2D 设备目标丢失时只丢弃并重建设备相关资源，播放状态不丢失。

## 5. 外观边界

本步骤使用完整、可辨认的中性浅色功能视觉，包含正常、悬停、按下、焦点、禁用和播放强调状态；它不是步骤 15 的 Foobox Basic 最终视觉。图标由代码绘制，不引入外部图片或字体。输出设备、媒体库、页面切换和齿轮的准确位置与图标仍按已批准路线在步骤 14 完成，不在本步骤猜测；播放顺序继续使用 foobar2000 自带入口，FooCrate 不再增加选择器。

## 6. 验收标准

- 面板可添加、删除、重新添加，foobar2000 不崩溃且不存在旧窗口回调。
- 本地可跳转音频的播放、暂停、上一首、下一首、停止、进度和音量真实生效并双向同步。
- 不可跳转流明确禁用进度拖动；未知长度不显示虚假时间。
- 快速切歌、拖动、失焦、关闭窗口和多实例不留下错误按下态或旧进度。
- Debug/Release 构建、状态模型测试、包布局测试和隔离实例启动检查通过。
