## 1. 已批准结构

CoverFlow 是由 FooCrate 打开的原生独立窗口，不替换主窗口的 Playlist Workspace，也不依赖 JSplitter、Spider Monkey Panel 或 Columns UI 自由布局。

```text
┌──────────────────────────────┐
│          CoverFlow           │
│   前一专辑   当前   后一专辑   │
├──────────────────────────────┤
│         Playlist View        │
└──────────────────────────────┘
```

上部显示可横向浏览的 CoverFlow，下部复用 FooCrate Playlist View；上下分隔比例可拖动并保存。

## 2. 已批准联动

- CoverFlow 中心专辑改变时，下方 Playlist View 定位、展开并滚动到对应专辑组。
- 下方曲目焦点进入另一专辑时，上方 CoverFlow 同步把该专辑移到中心。
- 当前播放、Playlist View 当前选择和 CoverFlow 中心专辑是三种独立状态，必须使用不同视觉反馈。
- 双击中心封面或下方曲目按后续批准的统一播放规则执行；单纯浏览不改变播放。
- 关闭窗口不停止播放；主窗口与 CoverFlow 窗口共享真实播放列表数据和封面缓存。

## 3. 实现前再决定

- 窗口默认尺寸、上下比例和最小尺寸；
- CoverFlow 的可见封面数量、透视、反射、阴影和动画时长；
- 鼠标滚轮、拖动、键盘、触控板与减少动画模式；
- 数据来源是活动播放列表、Default Playlist 还是可切换来源；
- 自动列表变化、缺图、同名专辑、多碟专辑与大列表虚拟化；
- 独立窗口按钮、任务栏、跨显示器位置恢复和可访问性；
- Direct2D 变换是否足够，还是需要 Direct3D；任何新运行时依赖都必须另行批准。

本规格当前只冻结结构和联动，不授权开始代码实现。
