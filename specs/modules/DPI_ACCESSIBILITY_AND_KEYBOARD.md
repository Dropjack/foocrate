# DPI、无障碍与键盘操作规格

- 版本：0.1
- 状态：已验收
- 对应任务：[`../../tasks/018-完善DPI无障碍与键盘操作/README.md`](../../tasks/018-完善DPI无障碍与键盘操作/README.md)
- 最后更新：2026-07-15

## 1. 目标

任务 18 不重新设计已经顺手的 Refrain，也不要求整体模仿 Foobox。目标是在 Windows 10/11 的显示缩放、文本放大和高对比度环境中保留当前布局与操作结果，并修复用户已经观察到的跨屏文字发虚。完整纯键盘路径和屏幕阅读器对象树延后到首版打包之后单独完成，避免在 1.0 打包前改变用户已有键位库和焦点模型。

## 2. 已有基础

- 主面板绘制和鼠标命中区域已经统一使用 DIP，运行时能够收到 `WM_DPICHANGED` 并重建 DirectWrite 字体资源。
- 主面板已有播放列表、Playlist Browser、Playback Queue、Album Cover Browser 和 Album Track List 的方向键、Home、End、Page、Enter 等局部键盘行为。
- 底栏已有可见焦点环，未处理的按键会继续交给 Columns UI/foobar2000 全局快捷键。
- 主题系统已经检测 Windows 高对比度，并使用系统窗口、文字、选择和焦点语义色。
- Preferences 使用原生 Win32 控件，已有 Tab Stop、系统字体和操作系统提供的基础可访问语义。

这些行为继续保留；任务 18 不改变播放、分组、封面、主题和设置项含义。

## 3. 已确认缺口

### 3.1 运行时 DPI 与文本缩放

- 主面板改变 DPI 后只重建文字资源，尚未同步执行滚动范围校正和 ESLyric 子窗口几何刷新。
- Now Playing、Playlist View 和 Dependencies 等 Preferences 子页没有完整处理父窗口跨显示器后的 DPI/字体重建。
- 主面板使用固定的 Segoe UI DIP 字号，显示缩放有效，但 Windows“使文本变大”不能完整传入自绘文字和相应行高。

### 3.2 键盘路径

- 当前 Tab 会立即离开整块 Refrain，没有执行用户在 Foobox 中依赖的专辑组折叠/展开行为。
- 完整的纯键盘区域导航、进度、音量、五星和上下文菜单路径需要结合 Foobox 行为与用户现有键位库另行设计；本任务不提前建立第二套冲突模型。

### 3.3 屏幕阅读器语义（首版后续）

- 主面板的大部分控件由 Direct2D 自绘。操作系统目前只能识别一个 Refrain 窗口，不能识别其中按钮、列表、五星、滑块、选中项、禁用状态和当前值。
- 自绘状态变化尚未向 Windows 辅助技术发送焦点、选择、值和名称变化通知。

## 4. 批准后实施的行为

### 4.1 DPI 与系统字体

1. 主面板发生运行时 DPI 变化时，按新 DPI 重建字体与设备相关资源，重新计算布局，校正所有虚拟列表滚动范围，并同步 ESLyric 子窗口；不得丢失选择、播放位置、分隔比例或当前工作区。
2. 所有 Refrain Preferences 子页处理父窗口 DPI 变化，重建系统消息字体并重新布局；不得丢失尚未 Apply 的草稿、当前页签或滚动位置。
3. 自绘文字以 Windows 系统消息字体为基准，保留 Refrain 已验收的相对字号层级；系统仅改变文本大小时，文字、行高和必要控件高度一起增长，而不是只放大字后裁切。
4. 继续使用现有最小窗口和响应式布局。空间不足时优先省略次要文本或出现已有滚动，不允许关键按钮互相覆盖。

### 4.2 本任务键盘行为

1. Playlist Workspace 中，Tab 按 Foobox 的可观察行为，在“全部专辑组折叠”和“全部专辑组展开”之间切换。
2. Tab 不改变播放、选择或排序；如果当前已经是混合折叠状态，第一次按下先折叠全部，下一次再展开全部。
3. 在 Album Cover Browser Workspace、设置窗口或 Playlist View 不具备可折叠组时，Tab 继续交给 Columns UI/Windows 默认焦点链。
4. 现有方向键、Home、End、Page、Enter、Space 和全局快捷键传递保持不变。
5. Shift+Tab、完整区域导航、键盘操作进度/音量/五星、菜单键以及屏幕阅读器对象树延后到首版打包后的独立任务，并以 Foobox 可观察行为和用户键位库为输入。

### 4.3 高对比度、焦点与图标清晰度

1. 高对比度继续使用系统语义色，并保留边框、图标、文字等非颜色提示。
2. 现有代码绘制图标在 DPI 变化后使用新的 Direct2D DPI 与像素尺寸重新渲染，不能复用旧屏幕的缩放位图；本任务不改变已经验收的图标造型。
3. Refrain 不新增装饰动画；用户主动开启的多封面轮播仍按播放信息功能运行，不被误当作装饰动画强制关闭。

## 5. 明确不做

- 不参考或复刻 Foobox 的设置与脚本实现。
- 不改变当前主界面比例、紧凑分组、封面规则、主题方案或设置结构。
- 不增加第二套快捷键配置；除已批准的 Tab 分组行为外，快捷键绑定仍由 foobar2000 管理。
- 不在首版打包前实现完整屏幕阅读器对象树或重新设计图标；两者互不依赖。
- 不在任务 18 新增触摸手势、CoverFlow、迷你播放器或性能压力工具。
- 不把放大最小窗口当作 DPI 修复。

## 6. 人工验收矩阵

1. 在 100%、125%、150% 和 200% 显示缩放下打开主面板和全部 Refrain Preferences；跨不同 DPI 显示器移动设置窗口，确认无裁切、错位和草稿丢失。
2. 使用 Windows 文本大小 100% 与 125% 检查中英文长标题、双行曲目、紧凑组头、设置标签和按钮。
3. 在 Playlist Workspace 按 Tab 折叠全部专辑组，再按 Tab 展开全部；确认选择、播放和排序不变。开启 Auto-collapse 时确认 Tab 不破坏自动折叠状态；切换到 Album Browser 后确认 Tab 不误改数据。
4. 开启 Windows 高对比度，核对文字、焦点、选择、播放、禁用和错误状态；关闭后恢复原主题。
5. 回归 foobar2000 全局快捷键、用户键位库、中文输入法、ESLyric、播放列表拖放和多封面轮播。

## 7. 通过标准

- 上述 DPI 与文本大小组合没有关键控件重叠、裁切、错误命中或状态丢失。
- Tab 在关闭 Auto-collapse 的 Playlist View 中稳定切换全部专辑组；其他场景不误改列表状态或用户快捷键。
- 高对比度和原主题均可读；Windows/foobar2000 快捷键及输入法不被无条件吞掉。
- Debug/Release、相关自动测试和隔离开发实例人工检查全部完成。
