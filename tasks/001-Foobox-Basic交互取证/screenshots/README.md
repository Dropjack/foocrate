# Foobox Basic 截图目录

请把用于交互取证的 Foobox Basic 原始截图直接放在本目录。截图会作为任务 001 的参考证据进入 Git，但截图本身不等于已经批准的产品决定。

## 建议命名

优先使用以下格式，方便按顺序核对：

```text
01-main-lyrics.png
02-main-track-info.png
03-left-sidebar.png
04-playlist-menu.png
05-cover-menu.png
06-bottom-controls.png
07-narrow-window.png
08-dark-or-other-state.png
```

不必严格使用上述名称；已有截图可以直接复制进来，Codex 会统一盘点并在任务文档中说明每张图证明了什么。

## 当前截图索引

| 文件 | 主要用途 |
| --- | --- |
| `主界面.jpg` | `Playlist Workspace`、`Top Bar`、`Playback Bar` 和右下歌词状态 |
| `主窗口可以调整边界但不能拖拽移动.jpg` | 固定区域拓扑与可调整分隔尺寸 |
| `信息板的主界面.jpg` | `Track Details View` 与 `Lyrics View` 共用右下位置 |
| `封面浏览器.jpg` | `Album Cover Browser Workspace`：左侧专辑封面网格和右侧专辑曲目列表；Foobox 的非 Album 模式仅作排除证据 |
| `独立设置页面.jpg` | 仅盘点 Foobox 可配置项；不作为 FooCrate 内嵌设置页要求 |

## 截图要求

- 尽量保留整个 foobar2000 窗口，不要裁掉标题栏和窗口边缘。
- 涉及菜单、悬停、右键、拖拽或异常状态时，单独截一张。
- 保留原始分辨率，不要先压缩、缩放或转成文档。
- 截图可能显示歌名、文件标签或播放列表名称；提交前请自行确认其中没有不希望进入 Git 的隐私内容。
