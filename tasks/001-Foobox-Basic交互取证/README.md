## 任务目标

把 Foobox Basic 中仅靠截图或脚本无法确认的具体位置、命令和状态变化整理成可实现、可验收的结论。Codex 先完成能由只读文件证明的部分，只把真正需要视觉观察或用户偏好选择的问题交给用户；所有结论写在本文件，避免再建立多份重复文档。

## 明确不做

- 不写 C++、不创建工程、不安装外部组件、不部署。
- 不修改或保存任何内容到 `D:\Dev\foobar2000`。
- 不把 Foobox 脚本结构当成 Refrain 架构，也不复制脚本代码。
- 不重新讨论已经确认的产品决定，除非新证据证明存在技术冲突。
- 不要求用户一次回答整张界面的所有细节；按区域分批确认。

## 取证方法

每项只使用以下四种状态：

| 状态 | 含义 |
| --- | --- |
| 已确认产品决定 | 用户已经决定 Refrain 怎么做 |
| 文件事实 | 可由参考版本、配置值或脚本直接证明 |
| 实际观察 | 用户或安全参考环境中实际操作确认 |
| 待确认 | 现有证据不足，尚不能写入最终规格 |

工作顺序：

1. Codex 只读检查 `.fth`、Foobox 配置和相关脚本，填入文件事实。
2. Codex 根据用户截图记录结构位置，但不在没有原图文件时伪造像素尺寸。
3. 对仍待确认的项目，每轮只询问一个区域或一组紧密相关行为。
4. 用户给出结果后，Codex 写入“实际观察”或“用户决定”，并说明是否改变产品规格。
5. 全部架构相关项目解决后，更新 `PRODUCT_SPEC.md`，由用户最终核对。

用户提供的原始截图统一存放在 [`screenshots/`](screenshots/README.md)。Codex 盘点后会为每张图记录观察区域、可直接证明的内容和仍不能由静态截图证明的交互。

| `D:\Dev\foobar2000\profile\foobox\script\js_panels\search.js` | 搜索提交后定位当前列表结果或建立媒体库搜索列表的规则 | B |
| `D:\Dev\foobar2000\profile\foobox\script\js_panels\jsplaylist.js` 及 `jsplaylist` 子目录 | 列与分组配置、选择/播放/队列状态、键盘、菜单、评分列和拖放入口 | C |
| `D:\Dev\foobar2000\profile\foobox\script\js_panels\infoArt.js` | 右上封面跟随与轮换、两类信息切换、五星命令、ESLyric 显隐、中键与自动切换、默认上下比例 | D、E |
| `D:\Dev\foobar2000\profile\foobox\script\js_panels\properties.js` | 右侧上下分隔条拖动、最小边界、比例持久化 | E |

已确认的关键默认值：

- 右侧上下分区默认比例为 `0.5`，Foobox 会保存拖动后的比例。
- 右侧组合面板参考最小尺寸为宽 `50`、高 `100`；这只是旧脚本下限，不直接作为 Refrain 的可用性标准。
- 顶部第二类歌曲信息默认每 `12` 秒切换一次，可关闭；用户此前决定 Refrain 第一套直接同时显示两条次要信息，因此不照搬该计时切换。
- Foobox 底部在窄窗口会整体隐藏媒体库、播放顺序、输出、静音、音量和左侧工作区按钮；Refrain 的窄窗口策略仍需另行批准。
