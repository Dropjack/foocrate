## 任务目标

先冻结 Foobox Basic 的固定信息架构、页面模式、区域命名和实现边界，再把区域内部行为的观察问题分派给后续对应模块。Codex 完成能由原始截图和只读文件证明的部分，只把真正需要用户选择的问题交给用户；所有结论写在本文件，避免再建立多份重复文档。

本任务不要求现在一次性决定每个按钮、菜单、字段、状态和生命周期。它们只有在会改变根界面结构时才属于步骤 01；其余内容进入步骤 06–18 的对应模块规格和验收。

## 明确不做

- 不写 C++、不创建工程、不安装外部组件、不部署。
- 不修改或保存任何内容到 `D:\Dev\foobar2000`。
- 不把 Foobox 脚本结构当成 Refrain 架构，也不复制脚本代码。
- 不重新讨论已经确认的产品决定，除非新证据证明存在技术冲突。
- 不要求用户一次回答整张界面的所有细节；按区域分批确认。

## 取证方法

每项只使用以下四种状态：

| 状态 | 含义 |
| --- | --- |
| 已确认产品决定 | 用户已经决定 Refrain 怎么做 |
| 文件事实 | 可由参考版本、配置值或脚本直接证明 |
| 实际观察 | 用户或安全参考环境中实际操作确认 |
| 待确认 | 现有证据不足，尚不能写入最终规格 |

工作顺序：

1. Codex 只读检查 `.fth`、Foobox 配置和相关脚本，填入文件事实。
2. Codex 根据用户截图记录结构位置，但不在没有原图文件时伪造像素尺寸。
3. 对仍待确认的项目，每轮只询问一个区域或一组紧密相关行为。
4. 用户给出结果后，Codex 写入“实际观察”或“用户决定”，并说明是否改变产品规格。
5. 全部架构相关项目解决后，更新 `PRODUCT_SPEC.md`，由用户最终核对。

用户提供的原始截图统一存放在 [`screenshots/`](screenshots/README.md)。Codex 盘点后会为每张图记录观察区域、可直接证明的内容和仍不能由静态截图证明的交互。

## 固定结构决定

- Refrain 不是自由布局编辑器。面板不能拖走、浮动、交换、嵌套、增删或移动到其他位置。
- 已定义的分隔线可以调整相邻区域的宽度或高度，并保存合法尺寸；调整尺寸不改变结构。
- `Native Window Frame`、`Top Bar` 和 `Playback Bar` 始终存在。
- 第一版界面文案使用英文；中文只用于项目文档解释。代码使用本节英文正式名称。
- Foobox 设置页截图不是 Refrain 页面结构要求。Refrain 设置统一进入 foobar2000 Preferences，底部齿轮直接打开对应页面。

已批准的可调整分隔线：

| 分隔线 | 允许的变化 | 明确禁止 |
| --- | --- | --- |
| `Playlist Browser` / `Playlist View` | 调整左右宽度并保存 | 交换、隐藏或把浏览器拖入中央 |
| `Playlist View` / `Now Playing Panel` | 调整左右宽度并保存 | 交换、浮动或把右栏停靠到其他边 |
| `Now Playing Header` / 右下视图 | 调整上下高度并保存 | 改变上下顺序或拆成独立窗口 |
| `Album Cover Browser` / `Album Track List` | 调整左右宽度并保存 | 交换、叠放或移动到其他页面 |

## 页面模式与区域图

### Playlist Workspace

```text
┌──────────────── Windows 原生标题栏 ────────────────┐
├────────────────── Top Bar ────────────────────────┤
│ Playlist Browser │ Playlist View       │ Now      │
│                  │                     │ Playing  │
│                  │                     │ Header   │
│                  │                     ├──────────┤
│                  │                     │ Lyrics / │
│                  │                     │ Track    │
│                  │                     │ Details  │
├────────────────── Seek Bar ───────────────────────┤
└───────────────── Control Bar ─────────────────────┘
```

固定左右顺序为 `Playlist Browser | Playlist View | Now Playing Panel`。右侧内部固定为上部 `Now Playing Header` 与下部 `Lyrics View`/`Track Details View`。竖向和横向分隔线只调整尺寸。

### Album Cover Browser Workspace

```text
┌────────────── Windows 原生标题栏 ──────────────┐
├────────────────── Top Bar ────────────────────┤
│ Album Source Selector                          │
├──────────────────────────────┬─────────────────┤
│ Album Cover Browser          │ Album Track List│
│                              │                 │
│                              │                 │
├────────────────── Seek Bar ───────────────────┤
└───────────────── Control Bar ─────────────────┘
```

固定左右顺序为 `Album Cover Browser | Album Track List`。这是只按 Album 浏览的独立页面模式，不是把封面视图从 `Now Playing Panel` 拖出来。页面级工具只保留 `Album Source Selector`；`Top Bar` 和 `Playback Bar` 在页面切换时保持不变。

截图像素只作为后续默认视觉的比例参考，不写死为运行尺寸。默认比例由步骤 15 随默认视觉批准；各区最小尺寸、窄窗口和不同 DPI 由步骤 17 专门批准。

### 封面浏览器范围裁决

参考中的 JS Smooth Browser 不只浏览专辑。其顶栏模式和文件事实包括：

- Album（simple/advanced）；
- Album Artist / Artist；
- Folder / Library Directory；
- Genre；
- Library 或 Playlist 数据来源；
- 过滤输入框、分组数量和当前播放摘要。

用户明确拒绝把这些通用模式带入 Refrain。最终裁决是：

- 页面：`Album Cover Browser Workspace`；
- 页面唯一维度：Album；
- 左侧：`Album Cover Browser`，内部主要内容为 `Album Cover Grid`；
- 右侧：`Album Track List`；
- 页面级工具只保留 `Album Source Selector`，显示 `Media Library` 或某个播放列表；
- Artist、Album Artist、Folder、Library Directory、Genre、通用过滤框和分类模式按钮全部排除。

这不是 Refrain 自建第二份媒体库。数据继续来自 foobar2000 媒体库或播放列表；左侧的原生 Album List 入口也继续保留，两者职责不同。详细愿景与执行方案保存在 [`Album Cover Browser 模块规格`](../../specs/modules/ALBUM_COVER_BROWSER.md)。
