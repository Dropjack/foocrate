## 任务目标

完成当前环境能够重复验证的短时稳定性、性能和生命周期检查；把必须依靠 1.0 后长期日常使用才能发现的问题登记为持续观察项，不虚构长期无泄漏结论。

## 中文程序逻辑

1. 先固定构建、实例、媒体来源和操作序列，取得可比较的资源基线。
2. 对模型使用大而确定的输入，对组件使用隔离实例的启动/空闲/播放/退出序列。
3. 每轮记录进程资源、耗时、返回状态和残留；比较重复轮次是否趋于稳定。
4. 同时检查代码中每项异步工作、缓存、回调、计时器、子窗口和图形资源的创建与释放是否对称。
5. 只有问题可以重复出现并定位到 FooCrate 时才修改代码；修复后重跑同一序列。
6. 当前环境不能证明的长期问题写入 1.0 后观察清单，不阻塞当前可自动验证部分。
