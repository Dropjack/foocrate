## 状态含义

| 状态 | 人类语言含义 |
| --- | --- |
| 待讨论 | 只有想法，范围尚未共同确认 |
| 规格核对中 | 已有中文规格，正在和用户逐条检查 |
| 可实现 | 用户已批准规格，可以开始写代码 |
| 实现中 | 正在实现完整可用功能，尚不能验收 |
| 实现完成待验收 | 构建、测试和开发实例检查已通过，等待用户人工验收 |
| 已验收 | 用户已明确确认行为符合规格 |
| 受阻 | 存在有证据的外部阻碍，文档中写明解除条件 |

## 新任务建立规则

1. 从 [`_模板/README.md`](_模板/README.md) 的结构建立新的编号目录。
2. 编号递增，目录名使用“编号-中文短名称”。
3. 一项任务只解决一个可以完整验收的纵向功能；不要用“前端一半、后端一半”拆出不可用任务。
4. 在写代码前写清中文程序逻辑和用户检查点。
5. 在实现完成后记录实际命令、结果、开发实例路径和人工验收步骤。
