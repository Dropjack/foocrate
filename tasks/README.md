## 推进规则

- 项目路线编号和提交标题以 [`TODO.md`](TODO.md) 为准；提交时省略两位路线编号。
- 用户可以说“开始/继续 + TODO 标题”指挥；上一步未验收并提交，不进入下一步。
- 新任务从 [`_模板/README.md`](_模板/README.md) 建立，必须包含中文程序逻辑、验证记录和人工验收。
- 每个任务只解决一个可独立构建、测试和验收的纵向功能，不提交空布局、假数据或半套行为。
- 每轮先检查 Git 状态并保护已有修改；参考目录只读，部署仅允许 `.local/foobar-dev` 和 `.local/foobar-test`。
