## 任务目标

把当前代码绘制的临时按钮图形替换成用户提供的正式图标，并让同一语义在主界面、菜单和后续独立窗口中保持一致。图标必须在深浅主题和不同 DPI 下清晰，不改变任何已有命令行为。

## 明确不做

- 本任务不实现 Mini Player 或 CoverFlow，只盘点它们未来会需要的入口图标。
- 不从 Apple Music 或其他产品直接复制受保护图标；参考只用于风格方向。
- 不用模糊缩放的低分辨率截图冒充正式资产。

## 中文程序逻辑

1. 用户提供单独 SVG、透明 PNG，或带编号说明的图标表。
2. Refrain 为每个图标建立稳定语义名称，不按按钮当前坐标或文件顺序识别。
3. 单色资产作为主题可着色遮罩；多色资产保留原色并逐主题检查对比度。
4. 程序按 DPI 缓存所需尺寸，Hover/Pressed/Disabled/Active 只改变已批准的颜色或背景状态。
5. 资源缺失或解析失败时构建失败并列出语义名，不在 Release 静默显示空按钮。
6. 设备丢失或主题变化只重建图形资源，不改变命令和用户设置。

## 可检查步骤

### 步骤 1：资产与语义核对

- 输入：用户图标文件及说明。
- 动作：生成完整映射表，识别缺失、重复、尺寸、透明度与许可问题。
- 产物：用户可读的图标清单和预览。
- 用户检查方法：逐项确认名称、用途和方向。
- 通过标准：没有靠猜测建立的映射。
- 状态：等待用户提供资产

### 步骤 2：原生资源接入

- 输入：已批准映射。
- 动作：转换、缓存、主题着色并替换现有绘制。
- 产物：Debug/Release 可用的正式图标。
- 用户检查方法：在四套主题与常用 DPI 逐个操作。
- 通过标准：清晰、对齐、状态完整且命令无回归。
- 状态：待开始
