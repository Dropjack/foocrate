## 任务目标

为全新 ESLyric 面板建立一套与 Refrain 协调的完整推荐默认，同时保证已有用户配置和之后的右键修改不会被 Refrain 反复覆盖。

## 明确不做

- 不直接读写 `foo_uie_eslyric.dll.cfg`。
- 不捆绑 ESLyric，不接管歌词源、歌词目录或网络行为。
- 未证明实例配置可安全区分全新/已有/升级三态前，不自动导入预设。

## 中文程序逻辑

1. 在隔离实例用 ESLyric 正式右键界面制作候选默认。
2. 通过 Columns UI 正式 `export_config/import_config` 导出和导入不透明实例配置，不解析私有字节。
3. 分别验证全新、已有用户设置和版本升级；只有能可靠识别全新实例时才允许首次默认应用。
4. 已有设置默认保留；主动重新应用必须明确确认。
5. 导入后用户继续用 ESLyric 右键修改，Refrain 不在启动时回写。
6. 运行时背景色同步继续独立工作，导入失败只降级推荐预设。

## 可检查步骤

### 步骤 1：三态接口验证

- 输入：干净 dev/test、已有 Foobox 风格配置、ESLyric 1.0.6.7。
- 动作：导出、跨实例导入、修改、重启和升级模拟。
- 产物：能否安全实现的证据和配置边界。
- 用户检查方法：核对三态可见结果，不检查二进制内部结构。
- 通过标准：不覆盖已有设置，失败可恢复。
- 状态：调查中

### 步骤 2：完整默认清单核对

- 输入：官方右键菜单、用户偏好与 Refrain 主题规则。
- 动作：逐项冻结布局、字体、双语行、背景、无歌词和搜索显示。
- 产物：用户批准的完整默认表。
- 用户检查方法：在 ESLyric 面板逐项对照。
- 通过标准：视觉项完整，歌词源和路径等私人数据明确排除。
- 状态：待开始

### 步骤 3：安全应用与回归

- 输入：批准预设与三态判定。
- 动作：首次应用、保留现有、主动重置、版本降级和背景同步。
- 产物：可发布的推荐默认行为。
- 用户检查方法：干净安装、已有配置、右键修改和重启。
- 通过标准：用户始终能自由修改且不被回写。
- 状态：待开始
