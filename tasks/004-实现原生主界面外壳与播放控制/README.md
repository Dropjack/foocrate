## 任务目标

让用户第一次把 `FooCrate` 作为 Columns UI 面板加入布局，并直接用它控制真实播放。面板只展示已经完整工作的底部控制，不用空框或假按钮冒充未来的播放列表、封面和歌词区域。

## 明确不做

- 不修改或隐藏 Windows 标题栏、foobar2000 菜单和主窗口框架。
- 不实现播放列表、封面、歌词、评分、设置和主题。
- 不猜测截图中尚未取证的左右工具按钮；播放顺序的选择界面留在对应后续步骤。
- 不部署到 C 盘日常安装、只读参考目录或 `foobar-test`。

## 中文程序逻辑

1. Columns UI 请求创建 `FooCrate` 时，组件建立一个普通 Win32 子窗口，读取 DPI，创建播放状态模型并订阅 foobar2000 播放回调。
2. 模型保存正在播放、暂停、是否可跳转、当前位置、长度、音量、自定义设备音量模式和当前鼠标交互；不复制播放列表或音频数据。
3. 点击按钮先检查当前状态，再在 foobar2000 主线程调用正式播放服务。进度拖动先预览，合法松开才请求跳转；取消则回到真实位置。音量拖动发出真实分贝值，静音调用正式切换。
4. foobar2000 回调确认状态变化后，模型更新并使窗口失效；Direct2D 下一次绘制只读取模型，不访问磁盘或网络。
5. 本步骤没有新增用户配置；播放器本身保存播放顺序和音量。FooCrate 面板实例由 Columns UI 布局保存。
6. 停止、未知长度、不可跳转、自定义设备音量、失焦、捕获丢失和窗口关闭都有明确状态；销毁时注销回调并释放资源，不允许晚到回调访问旧窗口。

## 用户人工验收

1. 启动 `.local/foobar-dev/foobar2000.exe`，确认 Components 中仍显示 `FooCrate 0.1.0`。
2. 在 Columns UI 的 Layout 配置或 Live editing 中，把一个开发用区域替换/插入为 `Panels > FooCrate`；不要在 C 盘日常安装操作。
3. 播放一首有明确时长的本地测试歌曲，依次点击 Previous、Play/Pause、Next、Stop，确认 foobar2000 播放状态和图标同步。
4. 播放时拖动上方进度条并松开，确认只在松开时跳转；再拖动后移出进度条松开，确认取消并回到真实进度。
5. 拖动右侧音量条、点击静音/取消静音，并从 foobar2000 其他音量入口修改一次，确认双方同步；在音量区域滚轮也应生效。
6. 播放网络流或其他不可跳转内容，确认进度条呈禁用状态、拖动不伪造跳转，未知总时长显示 `--:--`。
7. 把 FooCrate 面板删除后重新添加；可选再添加第二个实例，操作其中一个时另一个同步。快速切歌、暂停、停止后正常关闭 `foobar-dev`，确认无错误提示。
