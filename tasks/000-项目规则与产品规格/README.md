## 任务目标

建立一个可以跨很多轮继续、每一步都能由用户检查的项目基础。完成本任务后，项目应具备可靠的中文读取规则、Git 边界、只读参考证据、产品总规格和 UI/UX 目标，但仍不进入 C++ 实现。

## 明确不做

- 本任务不创建 Visual Studio/CMake 工程，不编译组件。
- 本任务不写任何用户功能的 C++ 阶段性代码。
- 本任务不修改、清理或部署到 `D:\Dev\foobar2000`。
- 本任务不替用户批准产品范围，也不把脚本推断当作已经观察确认的行为。

## 决策与未决问题

- 已决定：项目不采用 MVP 或不可用的阶段性功能作为交付方式。
- 已决定：面向用户的逻辑解释全部使用中文，C++ 实现必须有对应中文逻辑。
- 已决定：参考目录严格只读，日常 C 盘安装严格不接触。
- 已决定：纳入 Git 管理的 foobar2000 SDK 版本为 `2025-03-07`，保留完整可构建源码、工程与原许可证；官方包内的预编译 `.dll` 和 `.lib` 按二进制禁令继续忽略，需要的库由已管理的源码工程重建。项目不修改第三方 SDK 源码，构建产物输出到已忽略的构建目录。
- 已决定：Codex 的 Windows 中文文本读取规则放在 ASCII 英文的用户级和项目级 `AGENTS.md` 中；不向 `.codex/config.toml` 写入未受支持的编码配置项。
- 已决定：第一套主界面以 `foobox8 basic.fth` 及用户提供的两张截图为结构基准；保留 Windows 原生标题栏。
- 已决定：主界面为左侧播放列表管理、中间分组播放列表、右上封面与基本信息、右下 ESLyric/歌曲信息切换区和底部控制区。
- 已决定：媒体库入口使用 Foobox 当前配置同款的 foobar2000 Album List，不实现 JS Smooth Browser 或 ReFacets 替代品。
- 已决定：ESLyric 1.0.6.7 是批准的歌词依赖；鼠标中键在歌词和原生只读歌曲信息间切换，歌词双击由 ESLyric 自身处理全屏显示。
- 已决定：Biography 和内嵌频谱不属于产品范围。Refrain 不依赖频谱组件，也不在主界面提供频谱按钮。
- 已决定：Refrain 设置进入 foobar2000 Preferences，主界面设置按钮直接打开对应页面；配置需支持版本迁移并在升级时尽量保留。
- 已决定：评分沿用 Playback Statistics 的 `%rating%` 数据库写入方式，默认不写入文件标签。
- 已决定：第一套提供完整可用的默认配色；深浅色、强调色和完整颜色系统随后作为独立完整模块实现。
- 已决定：最终主要交付物为 `.fb2k-component`，不是独立 EXE；第三方组件在许可未核清前独立安装，不随包分发。
- 已决定：foobar2000 自身管理全局快捷键；Refrain 注册稳定命令，不建立第二套快捷键系统，升级时不得无故破坏用户绑定。
- 已决定：两个隔离实例当前只有 Columns UI；ESLyric 与 Playback Statistics 虽已批准，但仍需在后续环境任务中从正式来源分别安装并验证。
- 待观察：截图底部左右两组图标的每个实际命令、禁用状态、右键行为和快捷提示，必须在只读参考界面中逐项操作确认。
