## 2. 一句话产品定义

Refrain 是一个 Windows 10/11 x64、foobar2000 2.25.10、Columns UI 3.5.0 环境中的原生整合式根面板组件。它以 C++、Win32、Direct2D 和 DirectWrite 重建用户熟悉的 Foobox Basic 布局与日常交互，但不继承 JSplitter、Spider Monkey Panel、JScript Panel 等 JavaScript UI 宿主架构。

“重建”强调外观结构、可观察行为和操作习惯，不是翻译或复制 Foobox JavaScript。Foobox 参考脚本只能帮助确认字段、命令和状态，正式实现必须通过 foobar2000/Columns UI SDK 和原生 Windows 技术完成。

## 3. 产品与发布形态

### 3.1 产品是一个整合组件

- Refrain 对用户呈现为一个 Columns UI 根面板，而不是许多需要自行拼装的小面板。
- Refrain 自己管理主分栏、播放列表、封面与基本信息、底部控制区、设置和状态持久化。
- 它可以通过 Columns UI 的正式托管接口嵌入经批准的第三方面板，目前唯一需要嵌入的是 ESLyric。
- 最终主要发布物是 `.fb2k-component`，不是独立 EXE 安装器。
- 如完整根布局必须额外导入 Columns UI 布局文件，发布时可以附带，但不能自动覆盖用户已有布局；这一点需要在实现阶段通过 SDK 验证。

### 3.2 安装后的预期

安装 Refrain、Columns UI 及必要外部组件后，用户在 Columns UI 中选择或导入 Refrain 根布局，即获得规格中的整体外观。升级 Refrain 时应保留用户的快捷键、播放列表、列宽/列顺序、分组样式、分隔比例、右下视图选择和其他兼容配置；只有发生有记录的迁移冲突时才允许安全回退。

## 4. 第一套主界面的确定结构

精确参考是 `D:\Dev\foobar2000\themes\foobox8 basic.fth` 与用户在聊天中提供的两张 Foobox 主界面截图。`.fth` 是参考布局文件，不是可直接转换成 Columns UI `.fcl` 的实现模板；Refrain 要重建其效果。

### 4.1 顶部区域

- 保留 Windows 原生标题栏及系统最小化、最大化/还原和关闭行为。
- 不复刻 Foobox 隐藏 Windows 标题栏、把窗口按钮完全自绘进内容区的方案。
- foobar2000 的文件、编辑、视图、播放、媒体库、帮助等正式菜单仍然可用，菜单命令来自 foobar2000 服务。
- 当前播放摘要可以按参考排列，但不能破坏系统标题栏、键盘菜单和窗口拖动。

### 4.2 左栏：播放列表管理与入口

- 顶部为当前播放列表搜索和新增入口。
- 其下有筛选、媒体库入口和用户已有的普通/自动播放列表，并显示项目数。
- 当前活动播放列表按截图用强调色整行标记。
- 用户已有智能播放列表（例如 FavPop、FavScore 等）由 foobar2000 保存，Refrain 只显示和操作它们，不重建、改名或覆盖查询。
- 媒体库入口固定使用参考配置当前同款的 foobar2000 原生 Album List。不要改用 ReFacets，也不要因为开发者认为另一方案更先进而重写媒体库。
- Foobox 中的 JS Smooth Browser 不进入 Refrain Basic 主界面，也不作为运行依赖。

### 4.3 中栏：分组播放列表

- 中间是日常操作的主体，显示当前活动播放列表。
- 用户截图中可见的列包括：封面、序号、标题、艺术家、专辑、流派、等级、比特率、编码和时间。
- 分组标题显示专辑/艺术家摘要与年份；组内可显示封面。
- 需要支持普通与自动播放列表、分组、列配置、排序、多选、焦点、当前播放、队列、拖放、上下文菜单和键盘操作。
- 列宽、顺序、显示/隐藏、分组方式和播放列表主窗口样式必须持久化并支持版本迁移。

### 4.4 右上：封面、评分和基本信息

从上到下固定为：

1. 正方形封面；
2. 五星评分控件；
3. 歌曲标题；
4. 第一条次要信息：`歌手 | 专辑`；
5. 第二条次要信息：`编码 | 编码配置 | 比特率 | 采样率`。

Foobox 原版把后两种次要信息放在同一行，每 12 秒轮换。用户明确选择 Refrain 同时显示两条，因此不要实现定时轮换。标题不计入这“两条次要信息”。字段缺失、网络流、未播放和无焦点时的回退文案需在模块规格中定义。

封面与信息默认在播放时跟随正在播放曲目；停止播放时可以跟随当前列表焦点。最终跟随切换入口、封面多图切换、缺图和刷新行为还要通过参考操作确认。

### 4.5 右下：歌词与歌曲信息切换

- 默认状态嵌入 ESLyric 的 Columns UI 歌词面板，不独立打开另一个 Refrain 窗口。
- 双击歌词区域的事件交给 ESLyric，使其使用自己的全屏歌词功能；Refrain 不重复开发全屏歌词。
- 在右侧组合区域使用鼠标中键，可把右下歌词替换成 Refrain 原生只读歌曲信息列表；再次中键返回歌词。
- 歌曲信息视图按字段名—字段值展示当前曲目的元数据，支持滚动、选择和复制。它不是标签编辑器；编辑使用 foobar2000 正式 Properties 命令。
- 切换不得改变右上封面与基本信息，不得停止歌词搜索或播放；切换状态需要持久化。
- 参考 Foobox 脚本已经证明中键在 ESLyric 与属性列表之间切换；正式实现仍需通过 Columns UI 面板托管和真实 ESLyric 操作验收。

### 4.6 底部控制区

- 上沿是横跨主窗口的播放进度条，左边显示已播放时间，右边显示总时长或经批准的剩余时间。
- 下方中央按截图排列播放顺序、上一首、播放/暂停、下一首和停止。
- 左右两组功能按钮、静音和音量的位置、分组关系与截图一致。
- 设置齿轮打开 foobar2000 Preferences 中的 Refrain 设置页，不打开常驻的第三个主工作区。
- 截图只能证明图标位置，不能证明每个图标的完整命令、禁用状态、右键菜单和提示。实现前必须在只读参考中逐项观察，不能凭图形猜测。

## 5. 外部组件与功能归属

### 5.1 Columns UI

- 用户运行时基础组件：Columns UI 3.5.0 stable。
- 开发 SDK：Columns UI SDK 8.1.0，已在 `third_party` 中纳入 Git。
- Refrain 是 Columns UI 原生组件；不是 Default UI 主题，也不能直接导入 Foobox `.fth` 取代开发。

### 5.2 ESLyric

- 已批准依赖版本：ESLyric 1.0.6.7 x64。
- 用途：右下歌词显示、自动/手动歌词搜索和下载、歌词解析以及双击后的 ESLyric 自身全屏界面。
- ESLyric 是原生 foobar2000 组件并支持 Columns UI。它内部用 QuickJS 运行歌词源/解析脚本，不属于被禁止的 JavaScript UI 宿主。
- 特批边界：QuickJS 只能用于 ESLyric 的歌词检索与解析，不能用于 Refrain UI。
- Refrain 负责托管、布局、缺失提示和必要的视觉协调；歌词源、下载设置和内部升级仍归 ESLyric。
- 未核清再分发许可前，ESLyric 不打包进 Refrain，用户需要独立安装。

### 5.3 Playback Statistics 与评分

- 已批准依赖版本：Playback Statistics 3.1.10。
- 五星控件使用当前 Foobox 相同规则，调用 `Playback Statistics/Rating/1` 到 `5`，取消评分调用 `<not set>`。
- 自动播放列表继续查询 `%rating%`。Refrain 不需要知道 FavPop、FavScore 等列表名称或具体查询，只要正确写入同一评分数据源。
- 默认只写 Playback Statistics 数据库，不额外写音频文件的 `RATING` 标签，避免与用户当前行为不一致。
- `MOOD` 是不同概念，不能拿它替代评分；若以后需要保留 Mood 功能，必须单独确认字段写入与界面入口。

### 5.4 Biography 与频谱

- Biography 完全移出产品范围，既不原生实现，也不要求外部组件。
- 频谱不嵌入主界面、不作为 Refrain 依赖、不提供 Refrain 专用按钮、默认不启动。
- 用户有更专业的独立频谱软件。若用户自行安装 foobar2000 频谱组件，仍可通过 foobar2000 自身菜单打开；Refrain 无需拦截或隐藏正式菜单命令。

## 6. 设置、主题和快捷键

### 6.1 设置入口与保存

- Refrain 设置放进 foobar2000 Preferences，而不是主界面第三个独立页面。
- 主界面的齿轮按钮直接打开对应 Preferences 页面。
- 每项设置使用稳定 GUID、明确类型、默认值、合法范围、配置版本、迁移策略和恢复默认能力。
- Refrain 升级必须尽量保留布局、列、分组、分隔位置、右下视图、字体和颜色；发生无法迁移的冲突时，要记录原因并只重置受影响项。

### 6.2 快捷键

- 不建立 Refrain 自己的全局快捷键系统。
- 可执行命令应注册到 foobar2000 正式命令系统，用户继续在 foobar2000 Preferences 中手工绑定快捷键。
- Refrain 迭代不能更换已发布命令的稳定身份，避免用户升级后重新设置快捷键。
- 键盘焦点和文本输入不得吞掉 foobar2000 全局快捷键，中文输入法组合状态除外，需要按 Windows 规则处理。

### 6.3 主题

- 第一套先实现与用户截图相符、所有状态可辨且完整可用的默认配色。
- 深色、浅色、强调色和更完整的颜色系统仍属于产品目标，但作为后续独立完整模块，不阻挡主布局与核心交互的先行纵向实现。
- “后续模块”不代表允许 MVP、占位色或不可读界面。第一套默认配色本身必须可完整验收。
- 后续主题变化主要改变配色、字体和状态色，不任意改变已批准主布局。

## 7. 明确禁止和排除

- 不使用 JSplitter、Spider Monkey Panel、JScript Panel 或其他 JavaScript UI 宿主。
- 不复制 Foobox JavaScript 源码；只重建经观察确认的行为。
- 不修改 `D:\Dev\foobar2000` 参考目录。
- 不接触 C 盘日常 foobar2000。
- 不把 foobar2000 二进制、ESLyric、Playback Statistics 或其他闭源组件提交进仓库。
- 不提供独立 EXE 作为主要产品形态。
- 不实现 Biography、内嵌频谱、收音机、移动端、云同步、在线账户、音乐商店、解码器或 DSP。
- 不以“先做空布局”“以后再补行为”“先用假数据”的 MVP 方式交付。
