## 中文程序逻辑

1. 程序读取活动播放列表的稳定曲目快照，用已编译 title-format 规则为每项计算分组键；相邻且键相同的曲目形成一组，曲目索引仍是 foobar2000 的真实索引。
2. 每组保存起止索引、标题摘要、折叠状态和封面请求身份；视口由“组标题行 + 可见曲目行”组成，折叠只隐藏画面中的曲目，不删除或改写播放列表。
3. 组封面只为可见组异步请求，并用列表身份、分组身份和请求代次核对晚到结果；快速滚动、切换列表或关闭窗口后，旧图片不能覆盖新组。
4. 列定义保存稳定 ID、标签、标题格式、排序格式、对齐、可见性、顺序和宽度。绘制先按当前配置计算列矩形，只格式化可见曲目的可见列。
5. 表头单击请求排序，拖动边界调整宽度，拖动表头改变顺序，右键切换显隐和分组；所有会改变真实列表顺序的动作必须先检查锁定状态并建立 Undo。
6. 分组、列和默认折叠选项写入带版本的 Refrain 配置；读取旧版或损坏值时逐项校验，保留仍合法的用户设置，无法修复的项目回退到已批准默认值。
7. 活动列表增删、替换、重排或元数据变化时重建受影响的组和可见行；选择、焦点和当前播放始终继续由 foobar2000 提供。

## 最终人工验收

只在当前已经打开的 `.local/foobar-dev` 中检查；不要安装到 C 盘日常实例。

1. 确认默认列表出现 `Album (simple)` 组标题、左侧组封面和默认十列；有封面、缺封面、多碟与长文本都不应让行高跳动。
2. 单击组标题确认整组选中；Ctrl 单击切换整组、Shift 单击扩展范围；双击组标题折叠/展开。折叠后播放组内曲目，组标题应出现三角形播放标记；按 F2 应展开并定位该曲目。
3. 右击表头依次试 `Group by` 六个预置、`Collapse all`、`Expand all`、`Auto-collapse` 和 `Collapse groups by default`。切换分组后用 foobar2000 Undo，确认真实列表顺序可恢复。
4. 单击 Title/Artist/Album 等表头两次，确认升序/降序箭头和真实排序；拖动列边界改宽度，拖动普通列标题换序；右键隐藏/恢复若干列。Cover 必须保持最左，不能把全部文字列隐藏。
5. 打开 `Preferences > Display > Refrain > Playlist View`：复制一个内置分组和一列，修改自定义名称、表达式、对齐和宽度后 Apply；Cancel 另一组修改，确认未生效；输入非法表达式，确认 Apply 被拒绝；Reset Page 恢复默认。
6. 在普通列表调整布局后重启开发实例，确认分组、列显隐/顺序/宽度保留。按 F5 刷新可见组封面，快速滚动和切换列表，确认旧封面不覆盖新组且界面不阻塞。
7. 对自动/锁定且禁止重排的列表切换分组或按列排序，确认 Refrain 明确拒绝且不伪装成功。最后回归步骤 08：鼠标/键盘选择、双击/Enter 播放、右键正式菜单、Delete、空列表和外部增删仍正常。
