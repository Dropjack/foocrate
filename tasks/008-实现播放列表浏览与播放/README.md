## 任务目标

让 Refrain 中央区域成为可日常浏览和播放当前活动播放列表的原生 `Playlist View`，先完成不依赖分组和封面的基础纵向功能。

## 中文程序逻辑

1. 主窗口从 foobar2000 取得当前活动播放列表、曲目句柄、选择、焦点和真实播放位置；这些状态继续由 foobar2000 保存和修改。
2. 视图只格式化和绘制当前可见行，滚动不会读取不可见文件或封面；大列表不会因为预先生成全部文字阻塞界面。
3. 鼠标和键盘操作先换算为稳定列表身份与曲目索引，再调用正式选择、焦点和默认动作接口；右键菜单冻结当前句柄，避免列表变化后误操作同索引的新曲目。
4. 所有列表回调先更新/失效快照，再请求主窗口重绘；回调本身不嵌套修改列表。
5. 删除仅针对允许修改的普通列表，先建立 foobar2000 Undo 备份；自动或锁定列表保持不变。
6. 窗口销毁后停止接收和投递列表刷新；异常、空列表、无活动列表和失效文件都有稳定显示。

## 人工验收

1. 当前已启动的 `foobar-dev` 已加载新版 Refrain。打开一个有曲目的普通播放列表，确认中央出现 `# / Title / Artist / Album / Length`，右侧封面/歌词区与底部控制栏仍在。
2. 单击、Ctrl+单击、Shift+单击多行；确认选择块、细焦点框和当前播放行左侧强调条是三个可区分状态。
3. 双击一行播放，再选另一行按 Enter；确认播放目标正确。按 F2，确认滚动并定位到真实当前播放行。
4. 用滚轮和右侧细滚动条浏览长列表；把窗口调高/调矮，确认不会跳出列表范围、曲目文字不会画到底部控制栏上。
5. 依次检查 Up/Down、Page Up/Page Down、Home/End、Ctrl+方向、Space、Ctrl+A；确认键盘焦点和选择符合规格。
6. 在普通列表选若干行按 Delete，再用 foobar2000 的 Edit > Undo 恢复；确认只删除选择且可以恢复。对自动/锁定列表按 Delete，确认内容不变。
7. 右键未选中行、已选中的多行和列表空白处；确认分别得到正确对象的 foobar2000 正式菜单。菜单中只检查安全命令即可，不必执行永久删除文件。
8. 切换空列表、小列表和大列表，并从 foobar2000 其他入口增加/删除曲目；确认 Refrain 立即同步，无崩溃、旧行或错对象操作。
