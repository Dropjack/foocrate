# FooCrate 1.0.5 发布说明

FooCrate 1.0.5 修正 Playlist View 使用 Tab 全部折叠或展开后跳离搜索目标的问题。本版本保持组件、面板、命令、设置身份和配置格式不变，可直接从 1.0.0–1.0.4 升级，不需要重新导入 Columns UI 布局。

## 修正

- 活动播放列表存在有效焦点曲目时，Tab 使用该准确项目索引作为定位锚点，不再沿用折叠前已经失效的显示行号。
- 折叠全部专辑组后，将焦点曲目所属组头尽量放在 Playlist View 第三个可见内容行。
- 展开全部专辑组后，将同一准确曲目尽量放在第三个可见内容行。
- 多个搜索结果、同名曲目或同一文件重复出现时，以活动播放列表中的真实项目索引区分，不回退到搜索结果第一项或专辑第一首。
- 列表顶部、底部、不足一屏或可见容量不足三行时，滚动位置按合法范围夹紧，不制造空白。
- 没有有效焦点曲目时，才使用当前视口中央行所属曲目作为回退锚点。

## 验证

- `1.0.5-beta.1` 已完成 x64 Debug 和 Release 构建，两种配置各 14/14 自动测试通过。
- 自动测试覆盖第三行优先、列表顶部、列表底部、不足一屏、零可见容量和准确组头映射。
- 用户已确认搜索多个歌手后点击的准确曲目在折叠和展开时定位正确，并批准正式化为 1.0.5。
- 正式包按快速正式化路径仅重建 Release，不重复已经通过的 Debug 和全套自动测试。

## 依赖与升级

- Columns UI 仍是必要宿主。
- ESLyric 和 Playback Statistics 仍为独立可选组件，不包含在 FooCrate 组件包中。
- 本版本不新增设置字段，不改变现有布局、分组、列、主题或快捷键引用。
- 安装、升级、卸载和回退步骤见 `INSTALLATION_AND_UPGRADE.md`。
